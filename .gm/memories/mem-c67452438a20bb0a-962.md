## Resolved mutable: aro-private-mode-local-model-vs-hosted-api-conflict

WebFetch of https://hcompany.ai/holodesktop-cli confirms: holo-desktop-cli supports a --base-url flag pointing at any OpenAI-compatible local server, e.g. 'holo run --base-url http://localhost:8080/v1 ...' for llama.cpp at Q4 on a Mac. Documentation explicitly states: 'When using self-hosted inference, no API key is needed... Nothing -- no screenshots, no keystrokes, no app content -- ever leaves your machine.' Resolution: the P0/alpha build (Aro Private mode) must run holo-desktop-cli against a LOCAL llama.cpp server serving Holo3.1-35B-A3B Q4 GGUF, using --base-url, NOT the hosted Models API. HAI_API_KEY is not part of the alpha/P0 code path at all -- it remains stored (gitignored, unused) for a possible future opt-in hosted mode but must not be wired into the alpha executor. This directly changes holoiroh-holo-cli-bridge's implementation approach -- rescoping that row now.
